🗃️ 第二部分：数据模型设计

MySQL 四张核心表 + 字段说明

表1: User 用户

user
├── id INT PK AUTO_INCREMENT
├── username VARCHAR(50) UNIQUE NOT NULL
├── password_hash VARCHAR(256) NOT NULL
├── created_at DATETIME DEFAULT NOW()
└── updated_at DATETIME ON UPDATE NOW()

表2: conversation 会话

conversation
├── id INT PK AUTO_INCREMENT
├── user_id INT FK → user.id
├── title VARCHAR(100) 默认"新会话"
├── template_id INT FK → prompt_template.id (可空)
├── status ENUM('active','archived') DEFAULT 'active'
├── created_at DATETIME
└── updated_at DATETIME

表3: message 消息

message
├── id INT PK AUTO_INCREMENT
├── conversation_id INT FK → conversation.id
├── role ENUM('user','assistant','system') NOT NULL
├── content TEXT NOT NULL
├── metadata JSON (工具调用信息/Token用量等)
└── created_at DATETIME

💡 metadata 存 JSON: 如果这条是 tool_call，记录调了哪个工具、入参、结果

表4: prompt_template 提示词模板

prompt_template
├── id INT PK AUTO_INCREMENT
├── name VARCHAR(50) 如"客服默认" "售后专员"
├── description VARCHAR(200)
├── system_prompt TEXT NOT NULL (含 {placeholder})
├── temperature FLOAT DEFAULT 0.7
├── is_default BOOLEAN DEFAULT FALSE
└── created_at DATETIME

🔌 API 接口设计

方法 路径 说明 认证
POST /api/auth/register 注册 ❌
POST /api/auth/login 登录 → 返回 JWT ❌
GET /api/conversations 会话列表 ✅ JWT
POST /api/conversations 新建会话 ✅ JWT
POST /api/chat 发送消息 + SSE 流式回复 ✅ JWT
GET /api/conversations/:id/messages 获取历史消息 ✅ JWT
GET /api/templates 模板列表 ✅ JWT

核心接口详述: POST /api/chat

请求体：
{
 "conversation_id": 1,
 "message": "查一下订单 OR2024001 的状态",
 "template_id": null // null = 使用会话当前模板
}

响应 (SSE):
event: token   data: {"token": "您的"}
event: token   data: {"token": "订单"}
event: token   data: {"token": "已"}...
event: tool_call   data: {"tool": "query_order", "args": {"order_id": "OR2024001"}}
event: done   data: {"message_id": 42}
