💡 为什么智言能成为你的"王牌"？

面试官看重的不是项目多 fancy，而是：

  • 完整性 — 从前端到 AI 到部署，你全链路能打通
  • 深度 — 不是调个 API 就完事，而是理解了 Memory/Agent/RAG 的原理
  • 代码质量 — 结构清晰、有测试、有文档
  • 你自己的项目 — 每一行都是你写的，面试时能讲出细节

🤔 第一个问题：项目定位

智言的"AI客服"要做到什么程度？

A

轻量版 — 智能对话助手

用户注册登录 → 多会话管理 → AI对话（纯 LLM 问答）
覆盖：Prompt、Memory、Streaming
优点：快速出活，聚焦核心
缺点：缺少 Agent 和 RAG，深度不够

B

标准版 — AI 智能客服

在 A 基础上 + Agent 工具调用（查订单/查库存/退换货）+ 提示词模板切换
覆盖：Prompt、Memory、Agent、Tools、Streaming
优点：Agent 是面试高频考点，有工具调用场景
缺点：比 A 多一倍工作量

C

完整版 — 智能客服 + 知识库

在 B 基础上 + RAG 知识库（上传文档 → 向量化 → 检索增强问答）
覆盖：Prompt、Memory、Agent、Tools、RAG、VectorStore、Embedding
优点：RAG 是企业落地最广的 AI 技术，简历杀手锏
缺点：工作量大，需要学向量库（Chroma/FAISS）
